## 为什么要重构

aPulse 1.0 是用 Astro 4 快速搭建的，大量业务逻辑堆在页面组件里。随着功能增多，维护成本急剧上升。

### 关键数据

- **推文详情页**: 636 行 → 9 行（-98.6%）
- **照片墙 JS**: 167 kB → 17.5 kB（-89.5%）
- **API 路由**: 全部从 JS 迁移到 TypeScript

### 技术栈升级

| 技术 | 1.0 | 2.0 |
|------|-----|-----|
| Astro | 4.x | 5.x |
| React | 18 | 19 |
| TanStack Query | - | v5 |
| TypeScript | 部分 | 全部 |

重构的核心原则是**控制范围**，优先保证可用性而非完美。
